🚀 Quick start

Download SRBMiner-Multi from its official repo, then run one line with your crb1… address. CRB mines as the algorithm neuromorph.

⬇️ Download SRBMiner-Multiofficial releases · Windows / Linux / Hive OS
# pool - steady payouts (recommended for most miners) SRBMiner-MULTI --algorithm neuromorph --pool stratum.cereblix.com:3333 --wallet crb1YOUR_ADDRESS --password x # solo - whole 50 CRB blocks (use port :3334) SRBMiner-MULTI --algorithm neuromorph --pool stratum.cereblix.com:3334 --wallet crb1YOUR_ADDRESS --password x
🧮 All CPU cores are used by default. Name each rig with --worker RIGNAME so you can see them separately on the pool dashboard.

⚙️ Settings & config

The key flags. You can pass them on the command line, or put them in SRBMiner's config.txt / a .bat / .sh launcher.

--algorithm neuromorph # always this for CRB --pool stratum.cereblix.com:3333 # pool (:3334 = solo); pick a node from the list below --wallet crb1YOUR_ADDRESS # your CRB address - rewards go straight here --password x # any value --worker RIGNAME # label this rig on the pool dashboard --cpu-threads -1 # -1 = all cores (default); or a number to limit --disable-gpu # CRB is CPU-only; skip GPU probing
💡 A typical start.bat (Windows) is just one line: SRBMiner-MULTI.exe --algorithm neuromorph --pool stratum.cereblix.com:3333 --wallet crb1YOUR_ADDRESS --password x - save it next to the miner and double-click.

🐝 Hive OS - flight sheet

SRBMiner-Multi ships with Hive OS. Create a Flight Sheet and point the miner at CRB:

1. Wallet: add a wallet with your crb1… address (coin: pick any / "custom") 2. Pool: Configure in miner → custom → stratum.cereblix.com:3333 (RU: ru.cereblix.com:3333) 3. Miner: SRBMiner-Multi 4. Miner config → set: Algorithm: neuromorph Pool: stratum.cereblix.com:3333 Wallet: crb1YOUR_ADDRESS Password: x 5. Apply the flight sheet to your rig(s).
If your Hive uses the "Setup miner config" text box, the equivalent flags are: --algorithm neuromorph --pool stratum.cereblix.com:3333 --wallet crb1YOUR_ADDRESS --password x. Name rigs with --worker to track them on the pool dashboard. ⚠️ The farm proxy does not support SRBMiner yet (XMRig only) - run SRBMiner directly on each rig.

📈 Squeeze out more hashrate

NeuroMorph is memory-bound, so memory tuning matters most.

Large / huge pages - often the single biggest win (+20-70%). SRBMiner enables them automatically once the OS allows it. On Windows: secpol.msc → Local Policies → User Rights Assignment → Lock pages in memory → add your user, log out/in, then run SRBMiner as Administrator. On Linux: reserve them with sudo sysctl -w vm.nr_hugepages=1280.

Thread count - "all cores" isn't always fastest; on hybrid Intel (P+E) try the P-core count too and keep whatever gives more H/s (--cpu-threads N).

Big EPYC / Threadripper (multiple CCD / NUMA) - one instance per CCD / NUMA node, pinned, then sum the hashrate.
