💻 1 · Native CPU miner

The official one-file miner - many times faster than the browser, and the simplest "real" option. Download, run with your address; it finds the public node on its own.

🪟 Windows x64cereblix-miner 🐧 Linux x64cereblix-miner
# pool - steady payouts (best for laptops & normal CPUs) cereblix-miner -addr crb1YOUR_ADDRESS -node https://cereblix.com/pool/api # solo - whole 50 CRB blocks, a lottery (better on big rigs) cereblix-miner -addr crb1YOUR_ADDRESS
🐝 Hive OS rig? Flight Sheet → Custom miner, Installation URL https://github.com/CereblixCRB/cereblix/releases/latest/download/cereblix-hiveos.tar.gz, Pool URL https://cereblix.com/pool/api, your crb1… wallet.
🇷🇺 RU / CIS and cereblix.com is slow or blocked? Mine via the Moscow relay (no Cloudflare): -node https://ru.cereblix.com/pool/api (solo https://ru.cereblix.com/api).

⚡ 2 · XMRig - for big multi-core CPUs

Our official fee-free XMRig (developer donation removed, GPL source). On large CPUs it scales better than the native miner. It speaks Stratum - point it at ours, pool or solo.

🪟 Windows x64.exe · 9.9 MB 🐧 Linux x64binary · 4.3 MB 🐝 Hive OS.tar.gz · 1.7 MB 🍎 macOS · Apple Silicon.tar.gz · 3.0 MB
GPL source: xmrig-cereblix-src.tar.gz · mirror on the GitHub xmrig release. Intel Macs build from source (the binary is native arm64).
# pool - steady payouts xmrig-cereblix -o stratum.cereblix.com:3333 -a nm/1 -u crb1YOUR_ADDRESS -p x # solo - whole 50 CRB blocks (now shows steady feedback shares too; -p diff=N to pick difficulty) xmrig-cereblix -o stratum.cereblix.com:3334 -a nm/1 -u crb1YOUR_ADDRESS -p x
💡 Solo (:3334) now serves an easy auto-vardiff share target, so XMRig shows steady "accepted" shares + live hashrate just like the pool - while real blocks still pay the full 50 CRB. Choose your own difficulty with -p diff=N.
🧮 CPU cores. This build uses all your CPU cores by default. If it ever runs on fewer threads than your CPU has, set the count yourself: add -t N (N = number of threads) or --cpu-max-threads-hint=50 (percent of cores). On Hive OS leave Extra config arguments empty (algo nm/1 and all cores are automatic); to limit cores there add "max-threads-hint": 50 (JSON, not -t).
🚀 More hashrate. Three levers, biggest first. (1) Huge pages - often +20-70%, the single biggest win. XMRig uses them automatically, but on Windows grant the privilege once: secpol.msc → Local Policies → User Rights Assignment → Lock pages in memory → add your user, log out/in, then run XMRig as Administrator. On Linux reserve them: sudo sysctl -w vm.nr_hugepages=1280. The startup log should read huge pages 100%. (2) Thread count - "all cores" isn't always fastest: NeuroMorph is memory-bound, so on hybrid Intel (P+E) also try the P-core count (-t N) and keep whatever gives more H/s. (3) Big EPYC / Threadripper (several CCDs / NUMA) - run one instance per CCD / NUMA node, each pinned with --cpu-affinity, and sum the hashrate; a single process can lag across nodes. With huge pages and the right thread count our fee-free build lands roughly on par with SRBMiner after its 3% fee.
🇷🇺 RU / CIS. If shares drop out or the connection keeps timing out, use the Moscow relay (local route, no Cloudflare, same Stratum): pool -o ru.cereblix.com:3333 or solo -o ru.cereblix.com:3334 - everything else stays the same.

🔥 3 · SRBMiner-Multi - often the fastest

The popular SRBMiner-Multi (v3.3.9+) mines CRB natively as the neuromorph algorithm and is frequently the fastest on big CPUs. It is closed-source with a 3% dev fee (kept by the SRBMiner team, not us) - but the speed usually more than makes up for it.

⬇️ SRBMiner-Multiofficial releases (win / linux / hive)
# pool - steady payouts SRBMiner-MULTI --algorithm neuromorph --pool stratum.cereblix.com:3333 --wallet crb1YOUR_ADDRESS --password x # solo - whole 50 CRB blocks: use port :3334 · name a rig: --worker RIGNAME
🇷🇺 RU / CIS: use the Moscow relay - --pool ru.cereblix.com:3333. 🧮 All CPU cores are used by default. Track each rig on the pool dashboard - name them with --worker NAME to see them separately.
🚀 Large pages here too. SRBMiner gains a lot from huge / large pages - grant the same Windows privilege (or reserve Linux hugepages) as in the XMRig tip above; it enables them automatically once the OS allows it. The "one instance per CCD / NUMA node" trick also applies on very large CPUs.

🚜 Running a farm? Aggregate rigs with the proxy

Our fee-free xmrig-cereblix-proxy (Stratum proxy) lets a whole farm share one connection and one config: point the proxy at the pool, then point every rig at the proxy.

🪟 Windows x64xmrig-cereblix-proxy 🐧 Linux x64xmrig-cereblix-proxy
GPL source: xmrig-cereblix-proxy-src.tar.gz.
# 1. config.json - "simple" works with any miner version; big farms can use "nicehash" # ("nicehash" = ONE pool connection for ALL rigs - needs miner v1.1+ AND proxy v1.2+) { "mode": "simple", "pools": [ { "algo": "nm/1", "url": "stratum.cereblix.com:3333", "user": "crb1YOUR_ADDRESS", "pass": "x", "keepalive": true } ], "bind": [ "0.0.0.0:3333" ] } # 2. run the proxy xmrig-cereblix-proxy -c config.json # 3. point every rig at the proxy (not the pool): xmrig-cereblix -a nm/1 -o PROXY_IP:3333 -u crb1YOUR_ADDRESS -p x
⚠️ Upstream must be the pool (stratum.cereblix.com:3333, or RU/CIS ru.cereblix.com:3333). Earnings still track per address on the pool dashboard.
🛠️ Monitoring (built-in HTTP API), per-rig stats, TLS for remote rigs and a systemd service: see the README. The proxy also prints an update notice when a newer build ships (download only - disable with --no-version-check).

🖧 4 · Run your own node

A full node = independent chain validation and your own vote in the network, with no reliance on our servers. Optional, but it's the trustless way to mine.

🪟 Windows x64cereblixd 🐧 Linux x64cereblixd
# run the node and let it sync the chain cereblixd -datadir ./data # the native miner can mine straight to your node: cereblix-miner -addr crb1YOUR_ADDRESS -node http://127.0.0.1:18751/api

… and mine to your own node with XMRig

XMRig speaks Stratum, the node speaks HTTP getwork - so a tiny bridge, cereblix-stratum, sits between them. Run it against your node and you mine solo, fully trustless: every block pays 50 CRB straight to you, validated by your own node.

🪟 Windows x64cereblix-stratum 🐧 Linux x64cereblix-stratum 🍎 macOS arm64cereblix-stratum 🍎 macOS x64cereblix-stratum
# 1. your node (above) is running & synced # 2. bridge Stratum -> your node, with steady feedback shares + auto-vardiff cereblix-stratum -listen :3334 -pool http://127.0.0.1:18751/api -solo # 3. point XMRig at your local bridge xmrig-cereblix -o 127.0.0.1:3334 -a nm/1 -u crb1YOUR_ADDRESS -p x # pick your own difficulty: -p diff=50000 · watch jobs/shares/ms live: add -v to the bridge
-solo serves an easy feedback-share target with auto-vardiff (so XMRig shows steady "accepted" shares + live hashrate), tuned to your CPU and the network; real blocks still go to your node. Default difficulty = the pool's; set your own with -p diff=N. The -pool flag is just the getwork API to mine against - your node (…:18751/api) for trustless solo.
💡 Solo vs pool? Solo keeps the whole 50 CRB per block but is a lottery (better on big multi-core rigs). Pool pays a steady trickle proportional to your work, auto-paid to your address at 0.05 CRB (1% fee) - far less variance, better for laptops. On the pool your log shows "share accepted" (proofs at an easier target); the real reward lands as automatic payouts. Track it all on the pool dashboard.
